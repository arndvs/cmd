The four Claude Skills that automate the Sunday batch
These live in tools/skills/ and are invoked by name. ctrl+shft can run them on schedule or on-demand from a Linear issue.
1. idea-mine — runs Friday night. Reads log.md for the week, scans ventures/*/changelog.md and raw/ for new material, scans recent commits in linked repos. Outputs 10 candidate post ideas to content/ideas/inbox.md with pillar tags and difficulty estimates.
2. sunday-batch — runs Saturday night. Reads content/ideas/inbox.md + vault.md, picks 5 (2 reach, 2 trust, 1 conversion per the framework), drafts each one in content/drafts/ using the relevant prompt template, runs voice-check against each. You wake up Sunday to 5 drafts plus a one-paragraph rationale for each pick.
3. voice-check — lint pass. Reads a draft, compares against content/voice/samples.md and anti-samples.md. Flags: generic AI tone, hype words, missing hook, missing proof noun, too-long sentences, anti-pattern phrases. Outputs a diff with suggested edits, not a rewrite.
4. post-lint — pre-ship review. Checks: hashtag count and quality, character count for the preview cutoff (210 chars), broken links, client confidentiality (flags any mention of named clients), CTA presence, pillar tag.
The ctrl+shft integration
This is the elegant part. ctrl+shft already orchestrates agents against issues. So:

Create a Linear project: content-engine
Issues become recurring jobs:

[recurring] Friday 8pm — run idea-mine skill
[recurring] Saturday 10pm — run sunday-batch skill
[on-demand] Draft post about <topic>
[on-demand] Pre-lint <draft path>



Each issue body points at the relevant command folder paths. ctrl+shft picks up the issue, reads CLAUDE.md, navigates to the right context, runs the skill, commits the output back to the command repo, closes the issue.
You wake up to commits like content/drafts/2026-06-02-aws-saa-week-3.md with a clean draft, voice-checked, ready for a 5-min edit pass.

What lives on Cloudflare specifically
Workers — the publishing layer:

publish-post — receives a GitHub webhook when a draft is approved (moved from drafts/ to posts/), reads the markdown, posts to LinkedIn via Buffer/Typefully API, logs to D1, archives to R2.
engagement-poll — cron-triggered every 6hr, pulls LinkedIn analytics for shipped posts (via whatever scraper or API surface is available), writes back to D1, updates content/posts/<filename> with performance notes.
inbound-router — webhook receiver for recruiter DMs forwarded from Gmail (via Gmail filter → email forward → Cloudflare Email Routing → Worker). Parses the recruiter message, creates a wiki entry at wiki/recruiters/<name>.md, opens a Linear issue tagged inbound-recruiter.
idea-capture — endpoint you can hit from your phone (or a Shortcut). Posts JSON, appends to content/ideas/inbox.md. The "drop a thought anywhere" pattern.

D1 — the engagement database:

posts — one row per shipped post (date, pillar, type, hook, body hash, status)
engagements — time-series of impressions/likes/comments/profile-views per post
recruiters — inbound DMs, source, role, status, last touch
targets — your 30-company target list, with last-engagement-date per person you're tracking

This isn't a dashboard — it's the dataset behind the dashboard. The dashboard is markdown in atlas/, regenerated by Claude from D1 on demand.
