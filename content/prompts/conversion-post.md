# Conversion Post Prompt

> Goal: clear ask. Optimized for replies, inbound, and applications.

## When to use

1x per week. The post where you make it clear what you do and that you're available / building / hiring. Direct but not desperate.

## Characteristics

- **Case study format.** Show results, then make the ask.
- **Social proof.** Real outcomes for real clients/projects.
- **Clear CTA.** What should the reader do? DM, comment, visit a link.
- **Confident.** No apologizing for the ask.

## Prompt template

```
Write a conversion post for [platform].

Topic: [one-line topic from ideas/vault.md]

Pillar: [engineering / AI products / agent engineering]

Case study / proof: [specific project, client outcome, or shipped artifact]

Call to action: [what you want the reader to do — DM, comment, visit link]

Voice rules:
- Read content/voice/style-guide.md and content/voice/samples.md first
- Match the tone, rhythm, and vocabulary from the samples
- Check against content/voice/anti-samples.md before finalizing

Structure:
1. The problem — what the client/project faced
2. What I did — specific, not vague
3. The result — numbers, outcomes, before/after
4. The ask — clear, one sentence, no hedging

Constraints:
- Must include a specific outcome or metric
- CTA is one sentence max
- Don't oversell — let the result speak
- If using client work, anonymize appropriately
```

## Examples of good conversion topics

- "Built [X] for a client. Here's what happened." + DM CTA
- "This is what I'm building next. Want early access?"
- Portfolio piece walkthrough → "I'm available for [type] work"
- Open source launch → "Try it, break it, tell me what's missing"
