# cmd-cadence

Output "Read cmd-cadence skill." to chat to acknowledge you read this file.

## Purpose

Execute a cadence rhythm by reading the template, pulling relevant data from cmd, and generating a review document.

## Inputs

- **Frequency** (required) — one of: `daily`, `weekly`, `monthly`, `quarterly`, `yearly`

If no frequency is provided, ask the operator which cadence to run.

## Steps

### 1. Resolve paths

```
CMD_DIR="${CMD_DIR:-$HOME/cmd}"
```

### 2. Read the cadence template

Read `$CMD_DIR/cadence/<frequency>.md` for the review structure and questions.

### 3. Gather data

Pull relevant data based on frequency:

#### Daily
- Yesterday's entries from `log.md`
- Active ventures (directories in `ventures/` with recent log mentions)
- Today's priorities from `strategy/rocks.md` (current week's milestones)

#### Weekly
- This week's entries from `log.md` (last 7 days)
- Content shipped this week (files in `content/posts/` or `content/drafts/` modified this week)
- 5 review questions from `cadence/weekly.md`
- Carryovers from last week's review (if exists in `cadence/reviews/`)
- Progress against `strategy/rocks.md` milestones

#### Monthly
- This month's entries from `log.md`
- Strategic alignment check against `strategy/objectives.md`
- Pattern spotting: which ventures got attention, which were neglected
- Content published this month
- Key decisions made (scan `decisions.md` files across ventures)

#### Quarterly
- Full quarter's `log.md` entries
- OKR retrospective against `strategy/objectives.md`
- Rock review against `strategy/rocks.md`
- Kill-list review: what to stop doing
- Venture health check: active vs. stalled

#### Yearly
- Full year scan of `cadence/reviews/` for trends
- Theme review against `strategy/theme.md`
- Big bets evaluation
- What worked, what didn't, what to change

### 4. Generate the review

Write the review to `$CMD_DIR/cadence/reviews/YYYY-MM-DD-<frequency>.md`:

```markdown
# <Frequency> Review — YYYY-MM-DD

## Data period

<start date> → <end date>

## Activity summary

<log.md entries for the period, grouped by operation type>

## <Frequency-specific sections from the template>

<Answer each question from the cadence template using actual data>

## Ventures

| Venture | Activity this period | Status |
|---|---|---|
| <name> | <count> log entries | active / stalled |

## Priorities for next period

1. TKTK — operator fills in
2. TKTK
3. TKTK

---

*Generated by cmd-cadence on YYYY-MM-DD*
```

### 5. Handle empty data gracefully

If `log.md` has no entries for the period, don't fake data:

```markdown
## Activity summary

No log entries for this period. If work happened but wasn't logged,
consider running `/cmd lint` to check for logging gaps.
```

### 6. Update log.md

Append:

```markdown
[YYYY-MM-DD] cadence | Generated <frequency> review
```

## Rules

1. **Use actual data** — never invent activity or fabricate log entries
2. **Reference strategy** — connect activity to objectives and rocks where relevant
3. **Leave TKTK for opinions** — the AI fills in facts, the operator fills in priorities and judgments
4. **Previous reviews are context** — reference the last review of the same frequency if it exists
5. **No prescriptive tools** — don't mention specific apps. Reference cmd paths only

## Output

After generating the review, display a summary:

```
✓ Generated: cadence/reviews/YYYY-MM-DD-<frequency>.md
  Period: <start> → <end>
  Log entries: <count>
  Ventures active: <count>
```

Then display the review content inline so the operator can review immediately.
