# calendar/

Daily notes and journal. Human-authored, one file per day.

This is the unstructured capture layer — the place for narrative that doesn't fit a log entry, a cadence review, or a mission sidecar. What happened today, what you're thinking, what you noticed.

## Naming convention

```
YYYY-MM-DD.md
```

Example: `2026-05-27.md`

## What goes here

- Daily observations, reflections, decisions-in-progress
- Notes from conversations that aren't formal meeting transcripts
- Working thoughts that might become strategy or content later
- Anything you'd write in a journal at end-of-day

## What does NOT go here

- Structured cadence reviews → `cadence/reviews/`
- Machine-maintained activity log → `log.md`
- Meeting transcripts → `raw/transcripts/`
- Content drafts → `content/drafts/`
- Task tracking → your PM tool

## Rules

1. **Human-authored only.** AI does not write daily notes. AI may read them for context.
2. **One file per day.** No multi-day entries, no weekly rollups (that's what cadence is for).
3. **No required structure.** Write what you want. Freeform is the point.
4. **Don't backfill.** If you missed a day, you missed a day.
